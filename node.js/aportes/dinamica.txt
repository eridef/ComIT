En el ejemplo anterior vimos cómo se maneja la comunicación entre un cliente y servidor utilizando recursos (assets) estáticos.
Una página estática muestra información permanente; esto quiere decir que los documentos que están en el servidor no se modifican a no ser que alguien lo haga a mano y los vuelva a copiar. Para poder copiar archivos entre máquinas podemos utilizar un protocolo de transferencia llamado File Transfer Protocol (FTP). Por ejemplo: https://www.azimutglobal.com/es 
En cambio, una página dinámica se genera a partir de una petición desde el usuario, que nos va a poder enviar un dato y en base a eso nosotros podemos generar el nuevo documento con los resultados pedidos. Por ejemplo, podemos buscar en nuestra base de datos por el nombre de un producto, crear una lista de forma dinámica y escribirlo en formato HTML para mostrarlo del lado del cliente. Un ejemplo muy fácil es https://google.com.ar 
Para crear páginas dinámicas podemos utilizar varios lenguajes como pueden ser JavaScript, Python, Ruby, C#, Java, etc. Al llegar un pedido al servidor va a identificar que el archivo pedido está en alguno de estos lenguajes y le va a delegar la responsabilidad a ellos de crear la respuesta. Es decir que vamos a tener un archivo programado en un lenguaje particular y lo que vamos a hacer es crear una respuesta en formato HTML para que el browser lo pueda interpretar.
¿Dónde se ejecuta esto? Al llegar a la sección de Node.js y utilizar Express vamos a estar trabajando de esta forma. Esta tecnología trabaja con del lado del servidor, en lenguaje Java Script partiendo de un evento (solicitud de cliente), devolviendo una respuesta (archivo).
